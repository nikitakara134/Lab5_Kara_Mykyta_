1. Методологія EVMEarned Value Management - світовий стандарт проектного менеджменту.

SPISchedule Performance Index - Індекс виконання графіка. Показує відхилення від розкладу.

Schedule Performance Index (SPI) показує, наскільки ефективно команда CityTransit використовує час відносно плану у 6 місяців.

SPIІндекс виконання графіка = EVEarned Value - Освоєний обсяг / PVPlanned Value - Плановий обсяг

  • SPI > 1.0: Випередження графіка.
  • SPI = 1.0: Чітко за планом.
  • SPI < 1.0: Відставання від графіка.

CPICost Performance Index - Індекс виконання бюджету. Показує фінансову рентабельність робіт.

Cost Performance Index (CPI) показує, наскільки ефективно витрачається бюджет проєкту ($151,470).

CPIІндекс виконання бюджету = EVEarned Value - Освоєний обсяг / ACActual Cost - Фактичні витрати

  • CPI > 1.0: Економія бюджету.
  • CPI = 1.0: В рамках бюджету.
  • CPI < 1.0: Перевитрата бюджету.

* Розрахунки ведуться на базі 214 Story Points (SP)Story Points - абстрактна одиниця вимірювання складності та обсягу задач в Agile. із Лаб №3 та бюджету $151,470 із Лаб №4. Вартість 1 SP = ~$707.80.

5. Аналіз Отриманих Результатів CityTransit

Оцінка виконання графіка (SPISchedule Performance Index - показує відхилення від графіка.)

Динаміка розробки: Проєкт тривав 6 місяців і завершився рівно за графіком (SPIІндекс виконання графіка = 1.00).

  • Місяці 1-2 (Затримка): Команда відставала від графіка (SPIІндекс виконання графіка = 0.80 - 0.92). Це було викликано складністю налаштування архітектури бази даних (Scope 1Напрямок 1: Базова архітектура) та початковим збором вимог від міської влади.
  • Місяць 3 (Прорив): Після налаштування середовища розробка мобільного додатку та APIApplication Programming Interface - механізм взаємодії бекенду та додатку. пішла дуже швидко. Команда закрила 117 SPStory Points - бали складності задач і повністю наздогнала план (SPI = 1.00).
  • Місяці 4-6 (Стабілізація): Завдяки ефективному AgileГнучка методологія розробки - підхід з роботою короткими ітераціями (спринтами).-спринтуванню, проєкт витримував планові темпи і навіть випереджав графік у 5-му місяці (SPI = 1.02), що дозволило спокійно провести фінальне тестування (QAQuality Assurance - контроль якості (тестування).).

Оцінка ефективності бюджету (CPICost Performance Index - показує фінансову ефективність.)

Фінансова успішність: Проєкт завершився з високою рентабельністю та економією бюджету (CPIІндекс виконання бюджету = 1.06).

  • Місяці 1-2 (Перевитрати): На старті фінансові показники були нижчими за норму (CPIІндекс виконання бюджету = 0.88 - 0.94). Actual Cost (AC)Фактичні витрати перевищував Earned Value (EV)Освоєний обсяг - цінність створеної роботи через високі початкові інвестиції в інфраструктуру та ліцензії.
  • Місяці 3-6 (Економія): Починаючи з 3-го місяця, індекс CPIІндекс виконання бюджету стабільно тримався вище одиниці (1.03 - 1.08). Це стало можливим завдяки тому, що Backend-розробники використовували готові open-source бібліотеки для маршрутизації замість написання коду з нуля.
  • Фінальний результат: За планового бюджету $151,470 (PVPlanned Value (Плановий обсяг)), фактичні витрати (ACActual Cost (Фактичні витрати)) склали $143,000, зберігши команді понад $8,400.

Загальні висновки для PMProject Manager - керівник проєкту.

Управлінська методологія EVMEarned Value Management (Управління освоєним обсягом) чудово показала себе на проєкті CityTransit:

  • Управління ризиками: Завдяки щомісячному відстеженню, відставання на старті (Місяць 1) було одразу виявлено, що дозволило PMProject Manager-у перерозподілити ресурси та вирівняти ситуацію вже до 3-го місяця.
  • Модель Fixed PriceМодель ціноутворення з фіксованою вартістю контракту до початку робіт. виправдала себе. Ризики виконавця на старті (перевитрати) були перекриті високою швидкістю команди у другій половині проєкту.
